Scenario

You are a junior data analyst working on the marketing analyst team at Bellabeat, a high-tech manufacturer of health-focused products for women. Bellabeat is a successful small company, but they have the potential to become a larger player in the global smart device market. Urška Sršen, cofounder and Chief Creative Officer of Bellabeat, believes that analyzing smart device fitness data could help unlock new growth oppofiunities for the company. You have been asked to focus on one of Bellabeat’s products and analyze smart device data to gain insight into how consumers are using their smart devices. The insights you discover will then help guide marketing strategy for the company. You will present your analysis to the Bellabeat executive team along with your high-level recommendations for Bellabeat’s marketing strategy.

About the company

Urška Sršen and Sando Mur founded Bellabeat, a high-tech company that manufactures health-focused smart products. Sršen used her background as an artist to develop beautifully designed technology that informs and inspires women around the world. Collecting data on activity, sleep, stress, and reproductive health has allowed Bellabeat to empower women with knowledge about their own health and habits. Since it was founded in 2013, Bellabeat has grown rapidly and quickly positioned itself as a tech-driven wellness company for women.

By 2016, Bellabeat had opened offices around the world and launched multiple products. Bellabeat products became available through a growing number of online retailers in addition to their own e-commerce channel on their website. The company has invested in traditional advertising media, such as radio, out-of-home billboards, print, and television, but focuses on digital marketing extensively. Bellabeat invests year-round in Google Search, maintaining active Facebook and Instagram pages, and consistently engages consumers on Twitter. Additionally, Bellabeat runs video ads on Youtube and display ads on the Google Display Network to support campaigns around key marketing dates.

Sršen knows that an analysis of Bellabeat’s available consumer data would reveal more opportunities for growth. She has asked the marketing analytics team to focus on a Bellabeat product and analyze smart device usage data in order to gain insight into how people are already using their smart devices. Then, using this information, she would like high-level recommendations for how these trends can inform Bellabeat marketing strategy.

Data Cleaning and Manipulation Documentation

Tools Used

  • R Programming Language
  • Libraries: tidyverse for data manipulation and visualization.

Steps Taken to Clean and Prepare the Data

1. Data Import

  • Loaded daily_activity and sleep_day datasets using read.csv().

2. Initial Exploration

  • Explored datasets to understand structure and contents:
  • Used head() to preview the first few rows.
  • Listed column names with colnames().
  • Verified data types using str().

3. Checking for Missing Data

  • Verified the presence of missing values in both datasets using sum(is.na()).
  • Result: No missing values found.

4. Removing Duplicates

  • Checked for duplicate rows using distinct().
  • Removed duplicate rows from both datasets:
  • daily_activity <- daily_activity %>% distinct()
  • sleep_day <- sleep_day %>% distinct()

5. Ensuring Consistent Date Formatting

  • Reformatted dates to ensure consistency:
  • daily_activity$ActivityDate <- as.Date(daily_activity$ActivityDate, format = "%m/%d/%Y")
  • sleep_day$ActivityDate <- as.Date(sleep_day$SleepDay, format = "%m/%d/%Y")

6. Alignment of Column Data Types

  • Verified column data types and ensured alignment using str().

7. Filtering Invalid or Extreme Values

  • Removed rows with invalid or extreme outlier values:
  • Ensured all values in TotalSteps, TotalMinutesAsleep, and SedentaryMinutes were ≥ 0.
  • Used filter() to retain only valid data.

8. Merging Datasets

  • Merged daily_activity and sleep_day datasets on Id and ActivityDate using merge():
  • combined_data <- merge(sleep_day, daily_activity, by="Id")

9. Removing Redundant Columns

  • Dropped unnecessary columns, such as SleepDay, after merging datasets:
  • combined_data <- combined_data %>% select(-c(SleepDay))

10. Derived Metrics

  • Created new metrics to enhance analysis:
  • Sleep Efficiency: SleepEfficiency = TotalMinutesAsleep / TotalTimeInBed
  • Activity-to-Sedentary Ratio: ActivityToSedentaryRatio = TotalSteps / SedentaryMinutes

11. Summarization by User

  • Aggregated data to calculate average values per user:
  • user_summary <- combined_data %>% group_by(Id) %>% summarise(AvgSteps = mean(TotalSteps, na.rm = TRUE), AvgSleepMinutes = mean(TotalMinutesAsleep, na.rm = TRUE), AvgSedentaryMinutes = mean(SedentaryMinutes, na.rm = TRUE), AvgSleepEfficiency = mean(SleepEfficiency, na.rm = TRUE))

12. Verification

  • Verified cleaned data using summary statistics:
  • summary(combined_data) provided min, max, mean, and median values for key metrics.
  • Visualized relationships to confirm data consistency:
  • Scatterplots: TotalSteps vs. SedentaryMinutes, TotalMinutesAsleep vs. TotalTimeInBed.

13. Saving Cleaned Data

  • Exported cleaned datasets for further analysis:
  • Combined data: write.csv(combined_data, "cleaned_combined_data.csv", row.names = FALSE)
  • User summary: write.csv(user_summary, "user_summary.csv", row.names = FALSE)
